# Thoughts 2nd May

Think about the story for the study.

DNS is the last piece of the encrypted Internet.
DNS over TLS/HTTPS/QUIC are solutions to this.
Threat model, do you trust your resolver?
Running a resolver is only a cost.
Selling browsing habits of your clients is lucrative.
Then it does not matter if the connection is encrypted.
You live in EU where your ISP would pay big fines if they sell your data without consent.
What about the people in other locations?
Just pick a resolver that you trust then?
Maybe you trust it now, but that can change over time.

Introducing proxies.
Oblivious DNS, good idea but requires additional infra to work (.odns servers)
Oblivious DoH, good idea but theres a risk with collusion between the HTTPS forwarder and DoH server
How about sending the DNS queries over the anonymity network Tor?
Good idea, since the relays already exist and is run by volunteers.
But Tor is slow, three jumps around the world.
Maybe it is possible to reduce the number of jumps?
Good idea, but does that not break the security/privacy of Tor?
It depends on the threat model. In a two-hop circuit it is easier to correlate traffic, linking the user to the destination.
So a passive observer looking at the entry and exit could know where the client is going.
If the client is using DoH over Tor, then the passive observed would know that the client is querying a DoH server, but nothing about the query.
The DoH server only sees that the query is coming from the exit, so according our threat model, this tweak is fine for DNS.
Alright, reducing the number of hops might be fine, but if we are unlucky the relays could be in different corners of the earth.
Well Tor by default is selecting random relays with respect to bandwidth as well. This has positive effects on latency too.
We could also limit the selection of relays to certain geographical locations.
Let us pick an entry relay and an exit relay in our country to reduce the latency.
Would that not break the security/privacy of Tor?
Well since our threat model for anonymous DNS queries is treating the resolver as the adversary, it does not matter if the relays are within the same jurisdiction.
You could also move one of the relays to a neighbouring country if you want.

So we want to explain that tweaking Tor circuits does not negatively impact the client security or privacy if the adversary is the resolver (destination).
We are using the existing Tor network with a new threat model.

We want to compare approaches to anonymous DNS queries with regards to:
1. latency/performance
2. collusion
3. deployability
