Rows 1--3 in Table~\ref{tab:conf} present configuration variations using the
Tor configuration file (\texttt{torrc}). In this setup, it is not possible to
reduce the number of hops below three, nor to specify the geographical location
of the middle relay.

Rows 4--6 show equivalent configurations constructed with \texttt{carml} and
\texttt{stem}, enabling a comparison between Tor's default relay selection
algorithm and a custom approach.

Rows 7--8 demonstrate configurations where the middle relay is explicitly
specified in a three-hop circuit using \texttt{carml} and \texttt{stem}, a
capability not available through \texttt{torrc} alone.

Finally, rows 9--11 illustrate two-hop circuit configurations, which are only
achievable when using \texttt{carml} and \texttt{stem}.

For each configuration in Table~\ref{tab:conf} we perform the following steps 1000 times: 
(1) start DoHoT using the specified configuration.
(2) wait for a circuit to be established and a successful query responded.
(3) send a query for an A record using a unique and random non-existing domain.
(4) measure the time (ms) for the reply to return.
(5) stop DoHoT.
