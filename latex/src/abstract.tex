\begin{abstract}
    While encrypted protocols between end users and Domain Name System (DNS)
    resolvers are gaining adoption, direct use of popular protocols like
    DNS-over-HTTPS (DoH) allow resolver operators to link DNS queries to user
    identity (i.e., public IP address), posing a privacy threat. Several
    privacy-preserving solutions to this problem have been proposed with
    varying trade-offs in terms of query latency, privacy provided under
    different threat models, and deployability. In this paper, we revisit
    DNS-over-HTTPS-over-Tor (DoHoT), where DoH is used over circuits in the Tor
    network to protect end-users. By changing circuit configuration we show
    that the average query latency in DoHoT can be cut in half compared to its
    default configuration (from 134$\pm$110ms to 61$\pm$135ms), while still
    providing protection in a stronger threat model than other work around
    anonymous DNS and comparable ease of deployment (just use Tor). While
    minimizing query latency is essential to user experience, our results
    emphasize the importance of also considering different threat models and
    deployability when comparing solutions for anonymous DNS queries.

\end{abstract}
